# 登录跳转问题修复报告

## 问题描述
用户反馈登录成功后界面停留在登录页面不跳转。

## 根本原因分析

1. **登录成功后无导航逻辑**：登录页面只显示成功提示，没有主动跳转到主页
2. **后端响应格式不匹配**：后端返回 `admin` 和直接的 `accessToken`/`refreshToken`，但前端期望 `user` 和 `tokens` 对象
3. **API 地址配置问题**：`10.0.2.2` 是 Android 模拟器专用，iOS 需要使用 `localhost`
4. **管理员密码哈希不匹配**：数据库中的密码哈希不是 `admin123` 的正确 bcrypt 值
5. **认证中间件查询错误的表**：中间件查询 `users` 表，但登录使用 `admins` 表

## 修复内容

### 1. 前端登录页面导航 (`app/lib/screens/login_screen.dart`)
```dart
// 登录成功后主动跳转到主页
Future.delayed(const Duration(milliseconds: 800), () {
  if (mounted) {
    Navigator.of(context).pushReplacement(
      MaterialPageRoute(builder: (context) => const MainScreen()),
    );
  }
});
```

### 2. 前端 API 配置支持 iOS (`app/lib/config/api_config.dart`)
```dart
// 根据平台返回不同的开发环境地址
if (Platform.isIOS) {
  return iosDevBaseUrl; // http://localhost:3005/api/v1
}
return androidDevBaseUrl; // http://10.0.2.2:3005/api/v1
```

### 3. 前端 AuthService 兼容两种响应格式 (`app/lib/services/auth_service.dart`)
```dart
// 适配后端响应格式：admin 或 user，tokens 或直接的 accessToken/refreshToken
final user = UserModel.fromJson(userData['user'] ?? userData['admin']);
final tokens = userData['tokens'];
final accessToken = tokens != null 
    ? tokens['accessToken'] 
    : userData['accessToken'];
final refreshToken = tokens != null 
    ? tokens['refreshToken'] 
    : userData['refreshToken'];
```

### 4. 后端登录响应格式修复 (`backend/src/controllers/auth.controller.js`)
```javascript
res.json({
  success: true,
  message: '登录成功',
  data: {
    user: { id, username, email, avatar, role },
    tokens: { accessToken, refreshToken },
    expiresIn: '7d',
  },
});
```

### 5. 后端认证中间件修复 (`backend/src/middleware/auth.js`)
```javascript
// 查询 admins 表而不是 users 表
const admin = await query(
  'SELECT id, uuid, username, email, role, status FROM admins WHERE id = ? AND status = 1',
  [decoded.id]
);
req.admin = { ... };
```
