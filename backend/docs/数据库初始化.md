# 数据库初始化脚本说明

> LynShae Backend MySQL 数据库初始化指南

---

## 📁 文件位置

```
backend/
└── docker/
    └── mysql/
        └── init/
            └── 01_init.sql    # 数据库初始化脚本
```

该脚本会在 Docker 容器首次启动时自动执行。

---

## 🚀 使用方法

### 方式一：Docker 自动执行（推荐）

```bash
# 启动 MySQL 容器时会自动执行 init 目录下的 SQL 脚本
docker-compose up -d mysql

# 查看初始化日志
docker logs lynshae-mysql
```

### 方式二：手动执行

```bash
# 连接到 MySQL
mysql -h localhost -P 3306 -u root -p

# 执行初始化脚本
source docker/mysql/init/01_init.sql
```

### 方式三：使用部署脚本

```bash
# 一键部署会自动执行初始化
./scripts/deploy.sh
```

## 📊 初始化内容

### 1. 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS lynshae_db 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 2. 创建 15 张核心表

| 序号 | 表名 | 说明 |
|------|------|------|
| 1 | users | 用户表 |
| 2 | roles | 角色表 |
| 3 | user_devices | 用户设备绑定表 |
| 4 | devices | 设备表 |
| 5 | product_categories | 商品分类表 |
| 6 | products | 商品表 |
| 7 | product_skus | 商品 SKU 表 |
| 8 | orders | 订单表 |
| 9 | order_items | 订单商品表 |
| 10 | device_logs | 设备日志表 |
| 11 | articles | 文章表 |
| 12 | operation_logs | 操作日志表 |
| 13 | refresh_tokens | 刷新令牌表 |
| 14 | coupons | 优惠券表 |
| 15 | user_coupons | 用户优惠券表 |

### 3. 创建索引

- 主键索引（所有表）
- 外键索引
- 查询优化索引
- 唯一索引

### 4. 创建视图

```sql
v_product_overview    -- 商品概览视图
v_order_detail        -- 订单详情视图
v_device_status       -- 设备状态视图
```

### 5. 创建存储过程

```sql
sp_update_product_sales  -- 更新商品销量
sp_create_order          -- 创建订单
```

### 6. 初始化数据

#### 默认角色

```sql
INSERT INTO roles (name, code, level, permissions) VALUES

('管理员', 'admin', 90, '["product:*", "order:*", "user:view", "device:*"]'),
('普通用户', 'user', 10, '["product:view", "order:create", "device:bind"]');
```

#### 默认商品分类

```sql

('机器狗', 'robot_dog', 1, 1),
('配件', 'accessories', 1, 2),
```


## 🔧 修改初始化脚本

### 添加新表

在 `01_init.sql` 文件末尾添加：

```sql
DROP TABLE IF EXISTS your_table_name;
CREATE TABLE your_table_name (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    -- 其他字段
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='表说明';
```

### 添加测试数据

在文件末尾添加 INSERT 语句：

```sql
INSERT INTO products (spu, name, price, category_id, is_on_sale) VALUES
('SPU001', '测试商品 1', 99.99, 1, 1),
('SPU002', '测试商品 2', 199.99, 1, 1);
```

## 🗑️ 重置数据库

### 警告：这将删除所有数据！

```bash
# 方式一：删除容器和数据卷
docker-compose down -v
docker-compose up -d mysql

# 方式二：手动删除数据库
mysql -u root -p -e "DROP DATABASE IF EXISTS lynshae_db; CREATE DATABASE lynshae_db;"
mysql -u root -p lynshae_db < docker/mysql/init/01_init.sql
```

## 🐛 常见问题

### Q1: 初始化脚本没有执行？

**原因**: Docker 容器已经存在，init 脚本只在首次创建容器时执行。

**解决**:
```bash
# 删除容器和数据卷
docker-compose down -v

# 重新启动
docker-compose up -d mysql
```

### Q2: 字符集错误？

**确保**:
```sql
-- 检查数据库字符集
SHOW CREATE DATABASE lynshae_db;

# 应该显示：DEFAULT CHARACTER SET utf8mb4
```

### Q3: 外键约束失败？

**原因**: 表创建顺序错误或引擎不一致。

**解决**: 确保所有表使用 `ENGINE=InnoDB`。
