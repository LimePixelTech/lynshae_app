## 环境要求

### 最低配置
- **CPU**: 2 核心
- **内存**: 2GB
- **磁盘**: 20GB
- **系统**: Ubuntu 20.04+ / CentOS 7+

### 推荐配置
- **CPU**: 4 核心
- **内存**: 4GB
- **磁盘**: 50GB SSD
- **系统**: Ubuntu 22.04 LTS

### 软件依赖
- Docker 20.10+
- Docker Compose 2.0+
- Git
- Nginx (可选，用于反向代理)

## 一键部署

### 步骤

```bash
# 1. 克隆项目
git clone <repository-url>
cd backend

# 2. 执行部署脚本
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

### 脚本功能

部署脚本会自动完成：
1. ✅ 检查系统依赖（Docker, Git）
2. ✅ 创建必要目录
3. ✅ 复制数据库初始化脚本
4. ✅ 生成环境变量配置（.env）
5. ✅ 生成随机 JWT Secret 和数据库密码
6. ✅ 启动 Docker 服务（MySQL, Redis, App）
7. ✅ 等待服务就绪
8. ✅ 显示访问信息

### 部署后检查

```bash
# 检查服务状态
docker-compose ps

# 查看应用日志
docker logs lynshae-app

# 测试健康检查
curl http://localhost:3000/health
```

## Docker 部署

### 使用 docker-compose.yml

```bash
# 1. 配置环境变量
cp .env.example .env
vim .env  # 修改密码和配置

# 2. 启动所有服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 停止服务
docker-compose down

# 5. 重建服务
docker-compose down
docker-compose up -d --build
```

### 服务说明

| 服务名 | 容器名 | 端口 | 说明 |
|--------|--------|------|------|
| mysql | lynshae-mysql | 3306 | MySQL 数据库 |
| redis | lynshae-redis | 6379 | Redis 缓存 |
| app | lynshae-app | 3000 | Node.js 应用 |

### 数据持久化

数据卷映射：
- `./docker/mysql/data` → MySQL 数据
- `./docker/redis/data` → Redis 数据
- `./uploads` → 上传文件
- `./logs` → 应用日志

## 手动部署

### 1. 安装 Node.js

```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
nvm use 18

# 或使用 apt (Ubuntu)
apt install nodejs npm
```

### 2. 安装 MySQL

```bash
# Ubuntu
apt install mysql-server-8.0

# 初始化数据库
mysql -u root -p < ../database/lynshae_init.sql
```

### 3. 安装 Redis

```bash
apt install redis-server
systemctl enable redis
systemctl start redis
```

### 4. 部署应用

```bash
# 安装依赖
npm install --production

# 配置环境变量
cp .env.example .env
vim .env

# 启动应用
pm2 start src/server.js --name lynshae-backend
pm2 save
pm2 startup
```

## Nginx 配置

### 基础配置

```nginx
server {
    listen 80;
    server_name api.lynshae.com;

    # 日志
    access_log /var/log/nginx/lynshae-access.log;
    error_log /var/log/nginx/lynshae-error.log;

    # 客户端上传大小限制
    client_max_body_size 10M;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 上传文件
    location /uploads {
        alias /path/to/backend/uploads;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 负载均衡配置

```nginx
upstream lynshae_backend {
    least_conn;
    server 127.0.0.1:3000;
    server 127.0.0.1:3001;
    server 127.0.0.1:3002;
}

server {
    listen 80;
    server_name api.lynshae.com;

    location / {
        proxy_pass http://lynshae_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## HTTPS 配置

### 使用 Let's Encrypt

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d api.lynshae.com

# 自动续期
certbot renew --dry-run
```

### 手动配置 SSL

```nginx
server {
    listen 443 ssl http2;
    server_name api.lynshae.com;

    ssl_certificate /etc/ssl/certs/lynshae.crt;
    ssl_certificate_key /etc/ssl/private/lynshae.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location / {
        proxy_pass http://localhost:3000;
        # ... 其他配置
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name api.lynshae.com;
    return 301 https://$server_name$request_uri;
}
```

## 性能优化

### 1. Node.js 优化

```bash
# 使用 PM2 集群模式
pm2 start src/server.js -i max --name lynshae-backend

# 调整 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" pm2 start src/server.js
```

### 2. MySQL 优化

编辑 `docker/mysql/conf/my.cnf`:

```ini
[mysqld]
# 连接数
max_connections = 500

# 缓存
innodb_buffer_pool_size = 1G
query_cache_size = 64M

# 日志
slow_query_log = 1
long_query_time = 2
```

### 3. Redis 优化

编辑 `docker/redis/conf/redis.conf`:

```conf
# 内存限制
maxmemory 512mb
maxmemory-policy allkeys-lru

# 持久化
appendonly yes
appendfsync everysec
```

### 4. 应用层优化

- 启用 Redis 缓存
- 数据库查询添加索引
- 使用连接池
- 静态资源使用 CDN

    compress

## 备份与恢复

### 数据库备份

```bash
# 创建备份目录
mkdir -p /backup/mysql

# 备份脚本
cat > /backup/backup_mysql.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="/backup/mysql/lynshae_${DATE}.sql.gz"
mysqldump -h localhost -u root -p${DB_PASSWORD} lynshae_db | gzip > $BACKUP_FILE

# 删除 7 天前的备份
find /backup/mysql -name "*.sql.gz" -mtime +7 -delete
EOF

chmod +x /backup/backup_mysql.sh

# 添加定时任务
crontab -e
# 每天凌晨 2 点备份
0 2 * * * /backup/backup_mysql.sh
```

### 数据库恢复

```bash
# 解压并恢复
gunzip < lynshae_20260314.sql.gz | mysql -u root -p lynshae_db
```

### 完整备份

```bash
# 备份整个项目
tar -czf backend_backup_$(date +%Y%m%d).tar.gz \
    --exclude='node_modules' \
    --exclude='logs' \
    --exclude='uploads' \
    backend/
```

## 故障排查

### 1. 应用无法启动

```bash
# 查看日志
docker logs lynshae-app

# 检查端口占用
netstat -tlnp | grep 3000

# 检查环境变量
docker exec lynshae-app env
```

### 2. 数据库连接失败

```bash
# 检查 MySQL 状态
docker exec lynshae-mysql mysqladmin ping -u root -p

# 查看 MySQL 日志
docker logs lynshae-mysql

# 检查网络连接
docker network inspect backend_lynshae-network
```

### 3. 内存不足

```bash
# 查看内存使用
free -h
docker stats --no-stream

# 清理 Docker 资源
docker system prune -a
```

### 4. 磁盘空间不足

```bash
# 查看磁盘使用
df -h
du -sh */

# 清理日志
> logs/app.log

# 清理 Docker
docker system prune
```
