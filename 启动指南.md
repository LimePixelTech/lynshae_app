# LynShae 启动指南

## ⚠️ Docker 网络问题

由于网络原因，Docker 镜像拉取可能失败。建议使用**本地开发模式**启动。

## 🚀 方式一：本地开发模式（推荐）

### 1. 启动 MySQL 和 Redis（使用 Docker）

```bash
cd /Users/freakk/.nanobot/workspace/lynshae

# 只启动数据库服务
docker-compose up -d mysql redis
```

### 2. 启动后端服务

```bash
cd backend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env

# 启动开发服务器
npm run dev
```

后端将在 http://localhost:3005 启动

### 3. 启动 Web 管理后台（可选）

```bash
cd backend/web-admin  # 如果存在

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 4. 启动 Flutter 应用

```bash
cd app

# 启动模拟器
flutter emulators --launch Medium Phone API 36.1

# 获取依赖
flutter pub get

# 运行应用
flutter run
```

## 🐳 方式二：完整 Docker 部署

如果网络条件允许，可以使用 Docker 部署：

```bash
cd /Users/freakk/.nanobot/workspace/lynshae

# 配置环境变量
cp .env.example .env

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

## 🔧 故障排除

### Docker 镜像拉取失败

1. 检查 Docker Desktop 是否运行：
   ```bash
   docker ps
   ```

2. 修复 Docker 凭证配置：
   ```bash
   echo '{"auths":{}}' > ~/.docker/config.json
   ```

3. 重启 Docker Desktop

### 端口被占用

```bash
# 查看占用端口的进程
lsof -i :3005
lsof -i :3306
lsof -i :6379

# 停止占用端口的进程
kill -9 <PID>
```
